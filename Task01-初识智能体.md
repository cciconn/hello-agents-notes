# Task01：初识智能体学习笔记

## 学习内容
1. 什么是智能体：传感器感知环境，并自主的通过执行器采取行动
  1.1 传统视角下的智能体：从反射智能体到学习型智能体的发展
  1.2 大语言模型驱动的新范式：传统智能体的能力源于工程师的显式编程与知识构建，其行为模式是确定且有边界的；而 LLM 智能体则通过在海量数据上的预训练，获得了隐式的世界模型与强大的涌现能力，使其能够以更灵活、更通用的方式应对复杂任务
  1.3 智能体的类型：基于内部决策架构的分类；基于时间与反应性的分类（反应式智能体，规划式智能体，混合式智能体）；基于知识表示的分类（符号主义AI，亚符号主义AI，神经符号主义AI）
2. 智能体的构成与运行原理
  2.1 任务环境定义：PEAS，即性能度量，环境，执行器，传感器。部分可观察，确定性和随机性，多智能体环境。
  2.2 智能体的运行机制：Agent Loop：感知，思考（规划、工具选择），行动
  2.3 智能体的感知与行动：明确的交互协议规范LLM与环境之间的信息交换。Thought，Action，Observation
3. 动手体验：5分钟实现第一个智能体
4. 智能体应用的协作模式：作为高效工具或自主的协作者
  4.1 作为开发者工具的智能体：GitHubCopilot,Claude Code,Trae,Cursor
  4.2 作为自主协作者的智能体：单智能体自主循环，多智能体协作，高级控制流架构
  4.3 Workflow和Agent的差异：Workflow是让AI按部就班地执行指令，而Agent则是赋予AI自由度去自主达成目标

## 习题
1.
  - case A不算是智能体，它代表了计算机的性能和算力，只是工具
  - case B属于混合式智能体，它具备了检测到障碍物的反应性，也具备了通过何种方式避开的障碍物的规划性
  - case C属于规划式智能体，它能够在落子之前进行复杂的思考和规划，并基于胜利的目标找出最佳路径
  - case D属于混合式智能体，它具备遇到用户投诉的反应性，还能从查询订单、分析问题等多个维度思考，规划出问题解决方案并且安抚用户的情绪
2.
  - 部分可观察：智能体只能通过传感器获取实时的心率、血氧含量等数据，无法直接获取用户的全部状态；
  - 随机性：用户同样的行为可能受不同的环境干扰而产生不一样的结果；
  - 动态性：环境和用户的生理指标会在智能体决策期间持续发生变化；
  - 综上该任务环境是一个部分可观察、随机、动态的单智能体环境
3.
  - workflow：优点：处理A1会非常精准和高效；缺点：出现A2的情况必须人为干预
    Agent：优点：可以自主处理所有售后退款政策；缺点：如果要提高用户体验需要用到训练有素的llm，这样比较烧token，成本较高
  - 对于利润较低单价较低的快销品或者售后责任界定比较清晰的产品更适合workflow的方案，而对于利润较高单价较高的、售后责任界定不清晰的或者需要技术沟通的产品更适合Agent方案
  - 对于本电商售后处理方案而言，可以整合两种方案，先由Agent对接用户，通过沟通，筛选出退款条件清晰的用户直接转到workflow去处理，其他的由Agent理解退款政策、分析用户历史行为、评估商品状况后，自主决策是否批准退款
4.
  - 后续尝试通过AI coding来实现
5.
  - 医疗诊断助手智能体
    系统1:初步症状分类与分流；常见病症状匹配；快速匹配科室
    系统2:复杂诊断；治疗方案规划
    协同工作机制：快速响应与过滤（系统1主导），深度推理与决策（系统2主导），人机协同，最终达成目标
6.
  - 产生“幻觉”的原因：训练数据本身就包含大量错误或矛盾的信息，再者生成的结果是概率生成而非事实检索，因此就有几率出现“幻觉”
  - 不设循环限制：可能导致死循环或无限循环，将token消耗完
  - 仅用准确率来评估智能体是否智能是不够的，要多维度，比如有效性、效率、认知能力、人机交互和安全性、可靠性等方面综合评价
